Artefact 01 — Course Project

Seismic Fault Detection

A custom-built Java tool for automated identification of subsurface faults in 2D seismic reflection imagery — bridging petroleum engineering domain knowledge with applied data science.

Petroleum Engineering × Data Science Java 21 · Maven Image Processing Final Project Presentation

Project context

For my final assessment in petroleum engineering coursework, I delivered a presentation on the application of data science methods across the oil and gas value chain — covering upstream exploration, drilling operations, petrophysics, reservoir modelling, and downstream processing. To anchor the discussion in something tangible rather than purely conceptual, I built a working seismic fault detection tool in Java and used its live output as the closing worked example.

Seismic interpretation is one of the most data-rich and uncertainty-heavy stages of upstream exploration. Geophysicists traditionally identify faults manually — a slow, subjective process. Automated and ML-assisted fault detection is an active area of industry research because reducing interpretation time and inter-analyst variability translates directly into commercial value.

What the tool does

The application ingests a greyscale seismic reflection image, applies image-processing techniques to isolate discontinuity features against the layered sedimentary background, and produces two outputs: an annotated visualisation of detected faults and a structured analysis report quantifying each feature.

Figure 1 Raw seismic reflection input (left) alongside the annotated output (right). The tool identified 17 discrete fault features, ranging from large structural zones to localised discontinuities, each labelled and bounded for visual interpretation.
Figure 2 Structured analysis report generated alongside the annotated image — including summary metrics, a geological interpretation flag, and per-feature pixel coordinates and sizes for downstream use by an interpreting geophysicist.

Skills and capabilities developed

Cross-disciplinary integration

Translating petroleum engineering concepts — acoustic impedance contrasts, structural traps, fault geometry — into computational features detectable by code.

Java image processing

Working with 2D pixel data, region detection, and bounding-box annotation in a Maven-managed Java 21 project.

Technical communication

Presenting a workflow to a mixed audience, scaffolding from "what is a seismic image" through to applied machine learning use cases.

Independent learning

The petroleum engineering content was lecture-based; the tool's data science implementation was self-directed beyond the course material.

Critical framing

Designing the tool's output as a triage step — explicitly recommending further analysis — rather than overselling automated detection as a replacement for expert interpretation.

Domain awareness

Understanding that automated detection is only valuable if its output respects the geological plausibility of what's being detected.

Reflection

This was the first project where I felt my two degrees actually combined rather than ran in parallel. The petroleum engineering content gave me the vocabulary to know what was worth detecting; the data science side gave me the means to detect it. Building the tool also taught me that technical credibility in a cross-disciplinary project comes from honouring both fields — a fault detector that ignores geological plausibility is just an edge detector with extra steps.

For future iterations I would extend the tool with proper machine learning — a small convolutional model trained on labelled seismic data — add confidence scoring per feature, and validate against a ground-truth dataset rather than relying on visual inspection alone.
